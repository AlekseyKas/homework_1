# Инструмент Автоматизированного Мониторинга и Реагирования на Угрозы

## Описание

Этот скрипт представляет собой комплексный инструмент для анализа сетевой безопасности, который:

1. **Сбор данных**: Загружает логи Suricata из нескольких образцов данных
2. **Анализ IP-адресов**: Выявляет подозрительные IP по частоте алертов и типам активности
3. **Проверка через VirusTotal**: Проверяет подозрительные IP через API VirusTotal
4. **Анализ уязвимостей**: Выявляет потенциальные уязвимости через Vulners API и паттерн-матчинг
5. **Реагирование на угрозы**: Имитирует блокировку и мониторинг угроз с учетом CVSS scores
6. **Email уведомления**: Автоматически отправляет email оповещения о критических угрозах
7. **Отчетность**: Генерирует JSON и CSV отчеты с детальной информацией
8. **Визуализация**: Создает графики для анализа результатов

## Запуск скрипта

### 1. Настройка конфигурации

Создайте файл `config.json` в корневой директории проекта:

```json
{
  "virustotal_api_key": "ваш_ключ_здесь",
  "vulners_api_key": "",
  "api_timeout": 10,
  "max_ip_checks": 10,
  "log_level": "INFO",
  "email_enabled": false,
  "smtp_server": "smtp.gmail.com",
  "smtp_port": 587,
  "email_from": "your_email@gmail.com",
  "email_to": "admin@company.com",
  "email_password": "your_app_password"
}
```

### 2. Запуск скрипта

```bash
python3 threat_monitor.py
```

## Безопасность API ключей

- API ключи хранятся в конфигурационном файле `config.json`
- Файл `config.json` добавлен в `.gitignore` и не коммитится в репозиторий
- Никогда не коммитите конфигурационные файлы с реальными ключами
- Каждый разработчик должен создать свой локальный `config.json`

## Требования

- Python 3.8+
- Библиотеки: requests, pandas, matplotlib, json
- API ключ VirusTotal (обязательно)
- API ключ Vulners (опционально, для расширенного анализа уязвимостей)

## Логика реагирования

- **Критические угрозы**: IP с malicious детекциями в VirusTotal → блокировка + **email уведомление**
- **Критические уязвимости**: CVSS ≥ 9.0 → немедленное исправление + **email уведомление**
- **Высокорисковые уязвимости**: CVSS ≥ 7.0 → запланированное исправление
- **Средний риск**: CVSS ≥ 4.0 → мониторинг
- **Подозрительная активность**: >10 алертов от IP → мониторинг

## Email уведомления

Система поддерживает автоматическую отправку email уведомлений при обнаружении критических угроз:

- **SMTP сервер**: Gmail (по умолчанию), можно настроить любой
- **Уведомления отправляются для**:
  - Критических IP адресов (malicious в VirusTotal)
  - Критических уязвимостей (CVSS ≥ 9.0)
- **Настройка**: Включите `email_enabled: true` и заполните email параметры в `config.json`

### Настройка Gmail для отправки уведомлений

1. **Включите двухфакторную аутентификацию** в аккаунте Gmail
2. **Создайте пароль приложения**:
   - Перейдите в настройки Google Account
   - Security → 2-Step Verification → App passwords
   - Создайте пароль для "Mail"
3. **Заполните config.json**:
```json
{
  "email_enabled": true,
  "smtp_server": "smtp.gmail.com",
  "smtp_port": 587,
  "email_from": "your_email@gmail.com",
  "email_to": "admin@company.com",
  "email_password": "your_app_password_here"
}
```

### Тестирование email уведомлений

Для тестирования email функциональности:

1. **Настройте email параметры** в `config.json`
2. **Запустите систему** с тестовыми данными
3. **Проверьте логи** на наличие сообщений об отправке email
4. **Проверьте почтовый ящик** на получение уведомлений

**Примечание**: Email отправляется только для критических угроз. Для тестирования используйте IP адреса с высоким уровнем malicious детекций или уязвимости с CVSS ≥ 9.0.

## API интеграции

### VirusTotal API
- Проверка IP адресов на malicious активность
- Получение детальной информации о репутации IP
- Ограничение: 10 проверок за запуск (настраивается)

### Vulners API (опционально)
- Поиск уязвимостей по продуктам из алертов
- Получение реальных CVE с CVSS scores
- Детальная информация о severity и описаниях
- Fallback на паттерн-матчинг при недоступности API
